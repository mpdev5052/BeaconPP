# Privacy Policy – Beacon — D365 DevTools Chrome Extension

**Last updated: April 23, 2026**

This Privacy Policy describes how the Beacon — D365 DevTools Chrome Extension ("Beacon", "Beacon — D365 DevTools", "we", "our", or "the extension") handles information when you use it within Microsoft Dataverse and Dynamics 365 environments.

---

## 1. Overview

Beacon is a Chrome browser extension designed to enhance productivity within Microsoft Dataverse and Dynamics 365. It operates entirely within your browser session and uses your existing authenticated credentials to query and display Dataverse record data.

---

## 2. Data We Access

When you use Beacon on a Dataverse or Dynamics 365 record form, the extension may access:

- **Record data** visible on the currently open form (e.g., fields, IDs, entity types)
- **Your authenticated session** — Beacon uses your existing browser login credentials to make API calls to Dataverse on your behalf. It does not ask for, store, or transmit your username or password.

---

## 3. How Data Is Used

All data accessed by Beacon is used solely to power the extension's features, including:

- Displaying related events, history, and family/linked records
- Querying Dataverse using the Microsoft Web API on behalf of the logged-in user

No data is used for advertising, analytics, profiling, or any purpose beyond the functionality of the extension itself.

---

## 4. Data Storage

Beacon may store a minimal amount of data locally in your browser using Chrome's built-in local storage. This may include:

- User preferences or settings
- Cached record history to support the History feature

This data remains on your device and is never transmitted externally.

---

## 5. Data Transmission

Beacon communicates only with the Dataverse/Dynamics 365 environment you are already logged into in your browser. Specifically:

- API calls are made to your organisation's Dataverse instance using your existing authenticated session
- No data is sent to any third-party server, external database, or service operated by us
- No data leaves your organisation's Microsoft environment

---

## 6. Third-Party Sharing

We do not sell, rent, share, or disclose any user data or record data to any third party. Beacon has no backend server and does not collect data centrally.

---

## 7. Permissions

Beacon requests only the browser permissions necessary to function on Dataverse and Dynamics 365 pages. These permissions are used exclusively to read page content and make authenticated API requests to your Dataverse environment.

---

## 8. Data Security

Since Beacon relies on your existing authenticated Microsoft session and does not store or transmit data outside your organisation's environment, your data security is governed by your organisation's Microsoft 365 / Dataverse security policies.

---

## 9. Children's Privacy

Beacon is intended for use by business professionals within enterprise Dataverse environments. It is not directed at children under the age of 13, and we do not knowingly collect data from children.

---

## 10. Changes to This Policy

If we make changes to this policy, we will update the "Last updated" date at the top of this page. Continued use of the extension after changes are posted constitutes acceptance of the revised policy.
